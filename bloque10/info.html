I/O Redirection & Functions (Info)

I/O Redirection

There are always three default files open: stdin (the keyboard), stdout (the screen), and stderr (error messages). These and other open files can be redirected. Each file descriptor is a number: 0 for stdin, 1 for stdout, and 2 for stderr. Additional descriptors (3–9) can be used temporarily and restored later.

Redirecting input and output

Use '<' to feed a file as stdin and '>' to redirect stdout to a file (truncating). Use '>>' to append. Combine with '2>&1' or '&>' to redirect stderr together with stdout. Use pipes '|' to chain commands.

Functions

A function is a subroutine: a block of commands that can accept positional arguments ($1,$2,...) and returns an exit status (via return or last command exit). 'exit' inside a function exits the whole script; 'return' returns a status to the caller.

Redirección I/O

Siempre hay tres archivos abiertos por defecto: stdin (teclado), stdout (pantalla) y stderr (mensajes de error). Estos y otros archivos pueden redirigirse. Cada descriptor es un número: 0 = stdin, 1 = stdout, 2 = stderr. Se pueden usar descriptores adicionales (3–9) temporalmente y luego restaurarlos.

Redirigir entrada y salida

Usa '<' para usar un archivo como stdin y '>' para redirigir stdout a un archivo (sobrescribiendo). Usa '>>' para agregar. Combina con '2>&1' o '&>' para redirigir stderr junto con stdout. Usa pipes '|' para encadenar comandos.

Funciones

Una función es una subrutina: un bloque de comandos que puede recibir argumentos posicionales ($1,$2,...) y devuelve un estado de salida (con return o el estado del último comando). exit dentro de una función termina todo el script; return devuelve un estado al llamador.
