Bloque 8 — Variables y Caracteres Especiales (Info)

Special Characters & Variables (English)

What makes a character special?
If it has a meaning beyond its literal meaning, a meta-meaning, then we refer to it as a special character. Along with commands and keywords, special characters are building blocks of Bash scripts.

Table of Special Characters:

  • comments: #
  • command separator: ;
  • terminator in a case option: .
  • "dot" command: ./file — "dot" as component of a filename
  • partial quoting: ""
  • full quoting: ''
  • comma operator: \
  • escape (backslash): \\
  • backticks: `command`
  • null command: :
  • dollar sign: $

Variables

Variables are how programming and scripting languages represent data. A variable is nothing more than a label, a name assigned to a location or set of locations in computer memory holding an item of data.

Assignment examples:

var=value
var2="one two three"
var3=one\ two\ three

Referencing a variable:

echo $var
echo ${var}

Indirect referencing example:

var=value
value=hello
eval b=\$$var
echo $b

Variable types

Bash does not segregate variables by type; essentially they are character strings, but Bash permits arithmetic operations if the value contains only digits.

Use declare -i (or typeset -i) to force integer arithmetic:

$ n=6/3
$ echo "n = $n"
n = 6/3

$ declare -i n
$ n=6/3
$ echo "n = $n"
n = 2

Manipulating strings

Bash supports many string operations. Examples:

  • ${#var} — length of $var.
  • ${var:position:length} — substring (0-based).
  • ${var#Pattern} / ${var##Pattern} — remove shortest/longest match from front.
  • ${var%Pattern} / ${var%%Pattern} — remove shortest/longest match from back.
  • ${var/Pattern/Replacement} / ${var//Pattern/Replacement} — replace first/all matches.
  • ${var/#Pattern/Replacement} — replace prefix if it matches.
  • ${var/%Pattern/Replacement} — replace suffix if it matches.

Parameter substitution

Common forms:

  • ${parameter-default}, ${parameter:-default}
  • ${parameter=default}, ${parameter:=default}
  • ${parameter+alt}, ${parameter:+alt}
  • ${parameter?err_msg}, ${parameter:?err_msg}

Examples of use: provide default filenames, assign default values, test for presence or non-empty, abort with message, etc.

Variable categories

  • Local variables (visible only inside a function)
  • Environment variables (affect shell/processes)
  • Positional parameters ($0, $1, $2 ...)
  • Built-in variables and special shell variables

Arrays

An array stores multiple values; zero-based indexing.

$ my_array=( zero one two three four five )
$ my_array[6]=six
$ echo ${my_array[6]}
six
$ echo ${my_array[@]}
zero one two three four five six
$ echo ${#my_array[@]}
7

Built-in variables

  • $EDITOR, $HOME, $HOSTNAME, $IFS, $PATH, $RANDOM

Special shell variables

  • $? — return value (exit status)
  • $$ — PID of the script
  • $- — flags passed to the shell (set)
  • $! — PID of last background job
  • $* and $@ — all positional parameters

Caracteres especiales y variables (Español)

¿Qué hace que un carácter sea especial?
Si tiene un significado más allá de su sentido literal —un meta-significado— lo llamamos carácter especial. Junto con comandos y palabras clave, los caracteres especiales son piezas básicas en scripts Bash.

Tabla de caracteres especiales:

  • comentarios: #
  • separador de comandos: ;
  • terminador en una opción case: .
  • comando 'dot': ./file — 'punto' como componente en un nombre de archivo
  • comillas parciales: ""
  • comillas totales: ''
  • operador coma (escape): \
  • escape (barra invertida): \\
  • backticks: `comando`
  • comando nulo: :
  • signo de dólar: $

Variables

Las variables representan datos: un nombre (etiqueta) que apunta a una ubicación en memoria que contiene un dato.

Ejemplos de asignación:

var=value
var2="one two three"
var3=one\ two\ three

Referenciar una variable:

echo $var
echo ${var}

Referencia indirecta:

var=value
value=hello
eval b=\$$var
echo $b

Tipos y manejo

Bash no separa variables por tipo; esencialmente son cadenas de caracteres, aunque Bash permite operaciones aritméticas si el valor contiene solo dígitos.

Usa declare -i para forzar evaluación entera:

$ n=6/3
$ echo "n = $n"
n = 6/3

$ declare -i n
$ n=6/3
$ echo "n = $n"
n = 2

Manipulación de cadenas

Bash tiene muchas operaciones para strings (longitud, subcadena, borrado por patrón, reemplazo, prefijo/sufijo, etc.). Ejemplos: ${#var}, ${var:pos:len}, ${var#pattern}, ${var%pattern}, ${var//pat/repl}, ${var/#pat/repl}, ${var/%pat/repl}.

Sustitución de parámetros

Formas comunes: ${parameter-default}, ${parameter:-default}, ${parameter=default}, ${parameter:=default}, ${parameter+alt}, ${parameter:+alt}, ${parameter?err_msg}, ${parameter:?err_msg}. Se usan para valores por defecto, asignaciones condicionales y comprobaciones con mensajes de error.

Categorías de variables

  • Locales (visibles solo dentro de una función)
  • Variables de entorno (afectan el comportamiento del shell/procesos)
  • Parámetros posicionales ($0, $1, $2...)
  • Variables integradas y variables especiales del shell

Arrays

$ my_array=( zero one two three four five )
$ my_array[6]=six
$ echo ${my_array[6]}         # six
$ echo ${my_array[@]}         # zero one two three four five six
$ echo ${#my_array[@]}        # 7

Variables integradas

  • $EDITOR, $HOME, $HOSTNAME, $IFS, $PATH, $RANDOM

Variables especiales del shell

  • $? — estado de salida del último comando
  • $$ — PID del script
  • $- — flags pasadas al shell
  • $! — PID del último trabajo en segundo plano
  • $* y $@ — todos los parámetros posicionales
